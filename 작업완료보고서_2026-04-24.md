## 1. 프로젝트 개요

요양원 요양보호사의 음성 인수인계를 자동으로 처리하는 **법적 증거 인프라 B2B SaaS**.  
핵심 파이프라인: `마이크 음성 → POST /api/v2/ingest → Whisper STT → Gemini 구조화 → PostgreSQL WORM 원장 → Notion DB → 대시보드 표출`

## 2. 오늘(2026-04-24) 완료한 작업

### [SESSION] Gemini REST API 전환 + 보안 4대 결함 완전 봉쇄

#### 2-1. Gemini API 키 교체 및 연동 방식 전환
- **문제**: 구 키(`AIzaSy...`) 월간 할당량 소진(429) + gRPC SDK가 새 키 형식(`AQ.Ab8R...`) 거부
- **조치**:
  - `google.generativeai` gRPC SDK → `httpx` REST API 완전 전환
  - Google Cloud Console에서 `generativelanguage.googleapis.com` API 수동 활성화 (프로젝트 `696331551106`)
  - 새 키 `backend/.env` 적용

#### 2-2. E2E 파이프라인 검증 (Done Contract 충족)
- HTTP 202 응답 확인
- Gemini 구조화 JSON (`summary`, `urgent_note`, `full_refined`) 정상 반환
- WORM 봉인 `worm_sealed: true`
- Notion 인수인계 DB 큐 `notion_queued: true`
- `NOTION_HANDOVER_DB_ID` 오작동 수정 (구 ID `3b217a7c...` → 정확한 ID `34cdbdd0...`)

#### 2-3. 최상위 보안 감사 — 4대 치명적 결함 발견 및 봉쇄

| # | 결함 | 봉쇄 방법 |
|---|------|----------|
| 1 | API 키가 URL `?key=` 파라미터로 노출 → httpx INFO 로그에 평문 기록 | `x-goog-api-key` HTTP 헤더 방식으로 전환 |
| 2 | 재시도 로직 없음 → 429/503/Timeout 즉시 실패 | 지수 백오프 3회 재시도 (0s → 1s → 2s) |
| 3 | `httpx.post` 동기 호출이 async 핸들러 내부에서 이벤트 루프 블로킹 | `async with httpx.AsyncClient` + `await` 완전 전환 |
| 4 | `raw.lstrip("json")` 버그 — 'j','s','o','n' 개별 문자를 앞에서 제거 (정상 JSON 파괴) | `re.sub(r"^```(?:json)?\s*|\|\s*```$", "")` 정규식으로 교체 (2개 사이트) |

- 유출된 키가 담긴 `uvicorn_log.txt`, `uvicorn_err.txt` 삭제
- 수정 후 E2E 재검증: 로그 내 `?key=` 노출 **0건**, HTTP 202 정상

## 4. 현재 시스템 아키텍처 상태

```
[마이크 음성]
    ↓ POST /api/v2/ingest
[FastAPI — Cloud Run]
    ↓ OpenAI Whisper STT
[텍스트 변환]
    ↓ Gemini 2.5 Flash (REST API / x-goog-api-key 헤더)
[구조화 JSON: summary / urgent_note / full_refined]
    ↓ FORK A                          ↓ FORK B
[PostgreSQL WORM 원장]          [Notion 인수인계 DB]
[Cloud SQL — Append-Only]       [5대 마스터 DB 연동]
    ↓ SHA-256 해시체인
[증거 봉인 완료 — HTTP 202]
    ↓ SSE 실시간 Push
[대시보드 표출 — Firebase Hosting]
```
